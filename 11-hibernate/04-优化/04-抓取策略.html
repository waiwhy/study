抓取策略：可以改变sql语句的执行方式

查询一方有三种策略：
fetch="select(默认)|join|subselect"


1、查询一方的单条记录
默认情况下会使用 fetch="select" 策略
会执行多条sql语句

可以修改为 fetch="join" 策略
会执行左外连接的sql语句
public class FetchingStrategyTest {
    @Test
    public void test1() {
        
        Session session = HibernateUtil.openSession();
        Transaction tx = session.beginTransaction();
        
        Customer c1 = session.get(Customer.class, 14);
        System.out.println(c1.getOrders());
        
        tx.commit();
        session.close();
        
    }

}

2、查询一方的多条记录
如果想提高性能，会配置  fetch="subselect" 策略，使用子查询查询关联数据
<set name="orders"  cascade="all" fetch="subselect">
    @Test
    public void test2() {
        
        Session session = HibernateUtil.openSession();
        Transaction tx = session.beginTransaction();
        
        Query<Customer> query = session.createQuery("from Customer", Customer.class);
        List<Customer> list = query.list();
        for(Customer c : list) {
            System.out.println(c.getOrders().size());
        }
        
        tx.commit();
        session.close();
        
    }


查询多方有两种策略：
fetch="select(默认)|join"

配置：
<many-to-one name="customer" class="Customer" column="customer_id" cascade="all" fetch="join" />

注意：如果配置了join ，那么延迟加载将不会生效
