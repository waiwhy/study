延迟加载（按需加载）：为了减少数据库的访问效率，减少数据库的访问次数，提高查询性能。
机制：
在查询一个对象的时候，不会真正的执行数据库的查询
在需要对象的关联属性的时候，才去真正执行数据库的查询

get：默认是非延迟加载
load：延迟加载

load方法的默认延迟加载策略是 lazy="true"， 可以修改默认的延迟加载策略

一、属性级别的延迟加载

    <class name="Customer" table="t_customer" lazy="false">

测试：
public class LazyLoadingTest {
    @Test
    public void test1() {
        
        Session session = HibernateUtil.openSession();
        Transaction tx = session.beginTransaction();
        
        //Customer c1 = session.get(Customer.class, 14);//此句进行了数据库的查询
        //System.out.println(c1.getName());
        
        Customer c1 = session.load(Customer.class, 14);//此句没有进行数据库的查询
        System.out.println(c1.getName());
        
        tx.commit();
        session.close();
    }
}

二、类（关联）级别的延迟加载

关联级别的延迟加载是默认存在的，

1、一方（查询Customer是否延迟查询Order）

测试：（默认情况下hibernate关联级别具有延迟加载的特性）
    @Test
    public void test2() {
        
        Session session = HibernateUtil.openSession();
        Transaction tx = session.beginTransaction();
        
        Customer c1 = session.get(Customer.class, 14);//此句执行查询customer的sql
        System.out.println(c1.getOrders().size());//此句执行查询order的sql
        
        tx.commit();
        session.close();
        
    }

可以通过配置修改延迟加载的特性
<set name="orders"  cascade="all"  lazy="false">

2、多方（查询Order是否延迟查询Customer）

测试：（默认情况下hibernate关联级别具有延迟加载的特性）
    @Test
    public void test3() {
        
        Session session = HibernateUtil.openSession();
        Transaction tx = session.beginTransaction();
        
        Order o = session.get(Order.class, 17);//此句执行查询customer的sql
        System.out.println(o.getCustomer());
        
        tx.commit();
        session.close();
        
    }

可以通过配置修改延迟加载的特性
<many-to-one name="customer" class="Customer" column="customer_id" cascade="all" lazy="false" />
