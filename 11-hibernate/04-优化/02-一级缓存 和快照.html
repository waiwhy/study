一、一级缓存

Hibernate的一级缓存是指Session（属于事务范围的缓存，由Hibernate管理，无需干预），它是一块内存空间，用来存放从数据库查询出的java对象，有了一级缓存，应用程序可以减少访问数据库的次数，提高了性能。

在使用Hibernate查询对象的时候，首先会使用对象属性的OID值（对应表中的主键）在Hibernate的一级缓存进行查找，如果找到，则取出返回，不会再查询数据库，如果没有找到，再到数据库中进行查询操作。然后将查询结果存放到Session一级缓存中。





public class CacheLevelOneTest {
    /**
     * 测试一级缓存
     */
    @Test
    public void testCache() {
        
        Session session = HibernateUtil.openSession();
        Transaction tx = session.beginTransaction();
        
        //第一次查询（会查询数据库）
        Customer c1 = session.get(Customer.class, 14);
        System.out.println(c1);
        
        //第二次查询（不会查询数据库）
        Customer c2 = session.get(Customer.class, 14);
        System.out.println(c2);
        tx.commit();
        session.close();
        
    }
}
    /**
     * 测试一级缓存
     */
    @Test
    public void testCache2() {
        
        Session session1 = HibernateUtil.openSession();
        Transaction tx1 = session1.beginTransaction();
        
        //第一次查询（会查询数据库）
        Customer c1 = session1.get(Customer.class, 14);
        System.out.println(c1);
        
        tx1.commit();
        session1.close();
        
        
        Session session2 = HibernateUtil.openSession();
        Transaction tx2 = session2.beginTransaction();
        
        //第二次查询（会查询数据库）
        Customer c2 = session2.get(Customer.class, 14);
        System.out.println(c2);
        
        tx2.commit();
        session2.close();
        
    }

二、快照

当事务提交时，hibernate会调用 flush 方法，会清理一级缓存，也就是将内存中的数据与快照中的数据进行对比，如果不一致，则会执行同步操作，如果一致，则不执行update。

  • 快照是数据的副本
  • 快照属于一级缓存
  • 存储在堆内存中
  • 作用是保持数据的一致性

debug查看快照：Session -> PersistenceContext -> 

测试：
    /**
     * 测试快照
     */
    @Test
    public void testAutoUpdate() {
        
        Session session = HibernateUtil.openSession();
        Transaction tx = session.beginTransaction();
        
        //查询
        Customer c1 = session.get(Customer.class, 14);
        
        //修改
        c1.setName("张三");
        
        //执行更新
        //session.update(c1);
        tx.commit();
        session.close();
        
    }

三、一级缓存的管理

如果对象不在一级缓存中，数据库可不可以被自动更新。不可以。

把对象移出一级缓存的方法：
        session.evict(c1);
        session.clear();
测试：
    /**
     * 清理对象的一级缓存
     */
    @Test
    public void testEvictAndClear() {
        
        Session session = HibernateUtil.openSession();
        Transaction tx = session.beginTransaction();
        
        //查询
        Customer c1 = session.get(Customer.class, 16);
        c1.setName("张三1234");
        
        //session.evict(c1);
        session.clear();
                
        tx.commit();
        session.close();
    }
