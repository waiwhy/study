1、级联保存
当保存双向关系的一方时，默认会报告错误。此时应该在Customer中设置级联保存。
级联保存：操作一个对象是，同时操作其他的关联对象
在Customer端配置级联更新
<set name="orders" cascade="save-update">
测试用例中只保存Customer一方
    /**
     * 测试级联保存
     */
    @Test
    public void testCascateSave() {
        
        //构建数据
        Customer cust = new Customer();
        //cust.setId("1234567");
        cust.setName("Helen1234567");
        cust.setAge(18);
        cust.setGender('女');
        cust.setLevel("VIP");
        
        Order o1 = new Order();
        o1.setOrderno("20170928001");
        o1.setProductName("小米手机");
        
        Order o2 = new Order();
        o2.setOrderno("20170928002");
        o2.setProductName("小米笔记本");
        
        Session session = HibernateUtil.openSession();
        Transaction tx = session.beginTransaction();
        
        //处理数据之间的关系
        //建立一对多单向关联
        cust.getOrders().add(o1);
        cust.getOrders().add(o2);
        //o1.setCustomer(cust);
        //o2.setCustomer(cust);
        
        //保存数据（保存客户）
        session.save(cust);
        //session.save(o1);
        //session.save(o2);
        
        tx.commit();
        session.close();
    }


2、级联删除
如果不设置级联删除，那么在删除客户信息的时候，会把订单的customer_id外键设置为null
在Customer端配置级联删除
<set name="orders" cascade="save-update,delete">
也可以用all表示级联更新和删除
<set name="orders" cascade="all">
测试用例：
    /**
     * 测试级联删除
     */
    @Test
    public void testCascateDelete() {
        
        Session session = HibernateUtil.openSession();
        Transaction tx = session.beginTransaction();
        
        Customer customer = session.get(Customer.class, 13);
        session.delete(customer);
        
        tx.commit();
        session.close();
    }
