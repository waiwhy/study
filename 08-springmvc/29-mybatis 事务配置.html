在service实现层有一个saveUser的方法如下：
[java] view plain copy
  1. public int saveUser(User user){  
  2.         userDao.saveUser(user);  
  3.         userDao.saveUser(user);  
  4.     return 1;  
  5. }  
以上测试结果：第一个user保存成功，第二个user保存时主键冲突保存失败，但是第一个没有回滚（如果事务起作用会回滚的），为什么事务不起作用呢？？？经过许久的努力终于找到原因了（如果以下原因不对请不要喷！）。当加载spring-mvc.xml时，通过扫描com.cn.wolf包注入了一个没有事务的service，而pring的aop是在运行时拦截注入的，所以事务在此不起作用。
