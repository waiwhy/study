一、测试角色授权

1、修改shiro配置文件
    <!-- shiro主过滤器 -->
    <bean id="shiroFilter" class="org.apache.shiro.spring.web.ShiroFilterFactoryBean">
        ..........
        <!-- 用户访问未授权的资源时跳转的连接 -->
        <property name="unauthorizedUrl" value="/unauthorized.html"></property>
        
        <!-- authc：该过滤器下的页面必须验证后才能访问 -->
        <!-- anon：该过滤器的实现是空，任何用户都可以通过此过滤器 -->
        <property name="filterChainDefinitions">
            <value>
                /images/**=anon
                /css/**=anon
                /libs/**=anon
                /login.html=anon
                /sys/login=anon
                /kaptcha.jpg=anon
                /sys/user/list=roles[admin]
                /**=authc
            </value>
        </property>
    </bean>

2、创建 unauthorized.html 页面

3、修改自定义realm的授权方法
    /**
     * 授权
     */
    @Override
    protected AuthorizationInfo doGetAuthorizationInfo(PrincipalCollection principals) {
        
        System.out.println("授权");
        //分配权限
        return null;
        
    }

4、测试：
访问 /sys/user/list，用户浏览器跳转到 unauthorized.html 页面， 并且控制台打印 "授权"

二、为用户分配角色

1、修改自定义realm的授权方法
为用户添加相对应的角色
    /**
     * 授权
     */
    @Override
    protected AuthorizationInfo doGetAuthorizationInfo(PrincipalCollection principals) {
        
        System.out.println("授权");
        //分配权限
        
        //用户角色列表
        List<String> roleList = Arrays.asList("admin","user");
        SimpleAuthorizationInfo info = new SimpleAuthorizationInfo();
        info.addRoles(roleList);
        
        return info;
        
    }

2、测试：

访问 /sys/user/list，用户浏览器可以访问用户列表页面

三、测试资源授权

1、修改shiro过滤器列表
/sys/role/list=perms[sys:role:list]

2、测试：
访问 /sys/role/list，用户浏览器跳转到 unauthorized.html 页面， 并且控制台打印 "授权"


四、分配资源权限

1、修改自定义realm的授权方法
        List<String> permissions = Arrays.asList("sys:role:list","sys:role:info");
        //分配资源权限
        info.addStringPermissions(permissions);

2、测试：
可以访问受限的资源


五、使用注解的方式验证用户授权

1、屏蔽过滤器方式的用户授权
<!-- /sys/user/list=roles[admin]
 /sys/role/list=perms[sys:role:list] -->

2、开启注解
在spring的配置文件中配置
注意：一定要配置在 spring-mvc.xml  中
    <!-- 保证实现了Shiro内部lifecycle函数的bean执行 -->  
    <bean id="lifecycleBeanPostProcessor" class="org.apache.shiro.spring.LifecycleBeanPostProcessor"/>  
   
    <!-- 开启Shiro的注解(如@RequiresRoles,@RequiresPermissions),需借助SpringAOP扫描使用Shiro注解的类,并在必要时进行安全逻辑验证 -->  
    <!-- 配置以下两个bean即可实现此功能 -->  
    <!-- Enable Shiro Annotations for Spring-configured beans. Only run after the lifecycleBeanProcessor has run -->  
    <bean class="org.springframework.aop.framework.autoproxy.DefaultAdvisorAutoProxyCreator" depends-on="lifecycleBeanPostProcessor"/>  
    <bean class="org.apache.shiro.spring.security.interceptor.AuthorizationAttributeSourceAdvisor">  
        <property name="securityManager" ref="securityManager"/>  
    </bean>  
    


3、添加角色验证注解
    @RequestMapping("/list")
    @ResponseBody
    @RequiresRoles({"admin"})
    public DataGridResult getPage(@RequestParam Map<String, Object> params){

4、添加资源验证注解
    @RequestMapping("/list")
    @ResponseBody
    @RequiresPermissions({"sys:role:list"})
    public DataGridResult getPage(@RequestParam Map<String, Object> params){
